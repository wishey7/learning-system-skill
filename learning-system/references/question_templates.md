## 概念理解题

### 目的
测试用户对核心概念、定义、原理的理解程度。

### 模板结构
```
用自己的话解释：[核心概念/原理]
提示：[引导思考的关键点]
```

### 示例（以《反脆弱》为例）

**示例1**：
```
用自己的话解释：为什么"坚韧"不是"脆弱的反面"？塔勒布认为我们搞错了什么？
提示：想想"面对压力时，这三者的反应分别是什么？"
```

**示例2**：
```
用自己的话解释：反脆弱系统与脆弱系统的核心区别是什么？
提示：可以用"面对压力的反应"来说明
```

**示例3**：
```
塔勒布说"反脆弱是喜欢压力的"，这里的"压力"有什么前提条件？
提示：并非所有压力都能让系统变强
```

**示例4**：
```
解释"杠铃策略"的含义：为什么极端保守和极端冒险可以组合在一起？
提示：考虑风险的分布特性
```

**示例5**：
```
什么是"火鸡问题"？它揭示了什么思维陷阱？
提示：想想火鸡在感恩节前的"经验"
```

## 应用题

### 目的
引导用户将概念应用到实际生活或工作中，加深理解。

### 模板结构
```
[场景描述]，请用[概念]分析/解决。
提示：[应用方向提示]
```

### 示例

**示例1**：
```
你自己的生活里，有没有一个东西是反脆弱的？
提示：想想哪些事物在受到压力后反而变强了
```

**示例2**：
```
举一个现代社会"过度保护导致脆弱化"的例子，并说明如何让它变得反脆弱。
提示：可以思考教育、经济、健康等领域
```

**示例3**：
```
假设你是一名产品经理，如何让你的产品具备反脆弱特性？
提示：考虑用户反馈、市场竞争、技术迭代等方面
```

**示例4**：
```
分析你的职业发展：哪些方面是脆弱的？哪些是反脆弱的？
提示：可以从技能、收入来源、行业风险等角度思考
```

**示例5**：
```
设计一个"杠铃策略"应用于个人投资，具体如何配置？
提示：考虑极端保守和极端风险的组合比例
```

**示例6**：
```
如果你是一名管理者，如何让团队具备反脆弱性？
提示：思考组织结构、决策机制、容错空间等
```

## 延伸思考题

### 目的
挑战认知边界，引导用户进行反驳、批判或发散性思考。

### 模板结构
```
[争议性观点]，你同意吗？[追问]
提示：[思考角度提示]
```

### 示例

**示例1**：
```
塔勒布说现代社会系统性地消灭了反脆弱性，你同意吗？
举一个你观察到的例子。
```

**示例2**：
```
有人认为"过度保护是爱的表现"，用反脆弱理论反驳这个观点。
你会怎么说？
```

**示例3**：
```
如果反脆弱性如此重要，为什么大多数组织仍然追求稳定和可预测性？
这背后有什么深层原因？
```

**示例4**：
```
塔勒布批评现代教育体系"用考试扼杀反脆弱性"，你认为他的批评公平吗？
有没有教育的反例？
```

**示例5**：
```
思考"黑天鹅事件"与"反脆弱"的关系：
为什么反脆弱系统能从黑天鹅事件中受益？
```

## 问题设计原则

### 1. 层层递进
- 概念理解题 → 确保基础理解
- 应用题 → 深化实践理解
- 延伸思考题 → 挑战认知边界

### 2. 避免"是/否"问题
- ❌ 错误：反脆弱是脆弱的反面吗？
- ✅ 正确：用自己的话解释：为什么坚韧不是脆弱的反面？

### 3. 引导而非灌输
- ❌ 错误：杠铃策略是什么？
- ✅ 正确：解释杠铃策略的含义：为什么极端保守和极端冒险可以组合在一起？

### 4. 鼓励个性化回答
- 使用"你的生活"、"你的工作"、"你观察到的"等个性化表达
- 避免只有唯一正确答案的问题

### 5. 提供思考支架
- 通过"提示"字段给出思考方向
- 降低问题的认知负荷

### 6. 覆盖不同维度
- 每篇文章至少包含：
  - 2-3个概念理解题
  - 2-3个应用题
  - 1-2个延伸思考题
- 确保问题的多样性和覆盖面

## 使用指导

### 智能体生成问题时的注意事项

1. **参考文章内容**：问题应直接关联文章讲解的概念
2. **避免重复**：不同文章的问题应覆盖不同知识点
3. **难度适中**：根据用户的学习进度调整问题难度
4. **个性化调整**：根据用户的回答情况调整后续问题的设计
5. **保持开放性**：允许用户表达自己的理解和观点

### 问题数量建议

- 第一篇文章：5-6个问题（概念理解2个 + 应用2个 + 延伸思考1-2个）
- 后续文章：根据用户掌握情况，重点针对薄弱环节设计3-5个问题
- 总结文章：2-3个综合性问题，检验整体理解
