# 交互式学习系统工作流

## 任务目标
- 本Skill用于：系统性学习任意知识领域，通过迭代式文章生成、互动提问、反馈分析的闭环流程实现深度掌握
- 能力包含：学习项目初始化、文章生成与提问、用户反馈分析、掌握度评估、知识归档
- 触发条件：用户说"我想学习XX"、"帮我学习《XX》"、"我想系统掌握XX概念"等

## 前置准备
- 依赖说明：脚本使用Python标准库pathlib，无需额外安装
- 目录结构：首次学习新课题时，调用 `scripts/init_learning.py` 初始化学习项目

## 操作步骤

### 步骤1：初始化学习项目
- 执行脚本：调用 `scripts/init_learning.py` 创建学习项目目录
- 参数说明：
  ```bash
  python scripts/init_learning.py --topic "反脆弱"
  ```
- 创建的目录结构：
  ```
  learning_notes/反脆弱/
  ├── articles/           # 存放生成的文章
  ├── feedback/           # 存放用户回答
  └── progress.json       # 学习进度追踪
  ```

### 步骤2：生成第一篇文章与提问
- **智能体主导**：根据学习课题生成深入浅出的文章
- 文章结构要求：
  1. 基础概念解释（用通俗语言）
  2. 概念的重要性与价值
  3. 核心逻辑链（因果关系）
  4. 容易混淆的点与误区
  5. 实际应用场景
- 提问设计：在文章末尾生成5-8个问题，涵盖三类：
  - **概念理解题**（2-3个）：测试核心定义理解
  - **应用题**（2-3个）：结合生活实例应用概念
  - **延伸思考题**（1-2个）：挑战认知边界，反驳或发散
- 参考资源：使用 [references/question_templates.md](references/question_templates.md) 中的模板
- 保存文章：调用 `scripts/save_article.py` 保存
  ```bash
  python scripts/save_article.py \
    --topic "反脆弱" \
    --article-num 1 \
    --title "什么是反脆弱" \
    --content "文章内容..." \
    --questions "问题列表..."
  ```

### 步骤3：用户学习与反馈
- 用户行为：阅读文章，在文章末尾回答所有问题
- 用户告知：回答完成后说"我学完了"或"我已经回答了"
- 保存反馈：调用 `scripts/save_feedback.py` 保存用户回答
  ```bash
  python scripts/save_feedback.py \
    --topic "反脆弱" \
    --article-num 1 \
    --feedback "用户回答内容..."
  ```

### 步骤4：分析用户反馈
- **智能体主导**：深度分析用户的回答内容
- 分析维度：
  1. 概念掌握程度：哪些概念已理解、哪些模糊、哪些理解错误
  2. 兴趣方向：用户对哪个方向表现出更浓厚的兴趣
  3. 盲区识别：用户未提及或跳过的关键概念
  4. 认知深度：能理解→能解释→能举例→能反驳
- 检查进度：调用 `scripts/check_progress.py` 获取历史学习统计
  ```bash
  python scripts/check_progress.py --topic "反脆弱"
  ```

### 步骤5：迭代生成针对性文章
- **智能体主导**：根据分析结果生成下一篇针对性文章
- 文章特点：
  1. 开篇回应上一篇用户的回答（形成反馈闭环）
  2. 讲解用户掌握不足的概念
  3. 深入用户感兴趣的方向
  4. 纠正用户的理解偏差
- 保存文章：调用 `scripts/save_article.py` 保存新文章

### 步骤6：重复迭代
- 重复步骤3-5，直到用户满足掌握条件
- 迭代周期：通常3-5轮可完成一个课题的深度学习

### 步骤7：掌握度评估与归档
- **智能体主导**：评估用户是否已掌握课题
- 掌握标准（参考 [references/mastery_criteria.md](references/mastery_criteria.md)）：
  - 能理解所有核心概念
  - 能用自己的话解释概念
  - 能举出实际应用案例
  - 能回答反驳型问题
- 归档操作：
  1. 生成学习总结文档
  2. 提取关键知识点
  3. 建立知识链接（与相关概念建立关联）
  4. 输出最终笔记文件

## 使用示例：学习《反脆弱》

### 初始指令示例
```
我想学习塔勒布的《反脆弱》这本书的核心思想
```

### 第一篇文章生成要点
智能体将生成包含以下内容的文章：
1. 什么是反脆弱？（与脆弱、坚韧的区别）
2. 为什么反脆弱重要？（应对不确定性）
3. 核心逻辑：压力源、适度压力、过度保护
4. 常见误区：把坚韧等同于反脆弱
5. 生活实例：免疫系统、肌肉训练、市场经济

### 第一轮提问示例
**概念理解题**：
1. 用自己的话解释：为什么"坚韧"不是"脆弱的反面"？塔勒布认为我们搞错了什么？
2. 反脆弱系统与脆弱系统的核心区别是什么？请用"面对压力的反应"来说明。

**应用题**：
1. 你自己的生活里，有没有一个东西是反脆弱的？（提示：受到压力后反而变强的）
2. 举一个现代社会"过度保护导致脆弱化"的例子，并说明如何让它变得反脆弱。

**延伸思考题**：
1. 塔勒布说现代社会系统性地消灭了反脆弱性，你同意吗？举一个你观察到的例子。

### 反馈分析后的第二篇文章方向
根据用户回答分析，第二篇文章可能聚焦：
- 用户理解模糊的概念（如杠铃策略）
- 用户感兴趣的方向（如如何在工作中应用反脆弱）
- 用户未涉及的核心概念（如黑天鹅事件与反脆弱的关系）

## 注意事项
- 智能体主导内容生成：文章、提问、分析均由智能体完成，脚本仅负责文件管理
- 迭代式学习：不要一次性灌输所有知识，通过多轮互动逐步深入
- 个性化调整：根据用户的回答质量和兴趣方向动态调整内容
- 知识网络化：每个课题不是孤岛，要建立与相关概念的链接
- 避免填鸭式教学：通过提问引导用户思考，而非直接给出答案
- 及时反馈：每篇文章开篇回应上一轮用户的回答，形成对话感
